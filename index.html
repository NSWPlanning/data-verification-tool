Adding and maintaining Councils and their users

Creating a new Council

There are three steps:

  1. Create a new Council.
  2. Create a user account for the Council.
  3. Upload data files. This is covered in a later section called 'Manually upload Council data files'.

Create a new Council

  1. From any page, click on the Admin menu, then on 'Create Council'.
  2. Fill out the form. For most Councils, you will only need to fill out the name of the Council. Because the Council name can be different in four different ways, there is the ability to configure all instances of it.
  3. Once you're happy, click 'Create LGA'. If you make a mistake, don't worry. You can always come back and change these values later. See the section 'Edit a Council definition'.

Create a user for the Council

  1. From any page, click on the Admin menu, then on Users.
  2. Click on 'Create new user'
  3. Fill in the new user form. There are some things you need to be particularly aware of:
  4. The email address must be valid. It is used to notify the user of the success or failure of data uploads.
  5. Normal users should not have the 'Admin' box ticked. It will give them access to all data, and the ability to change other people's details.
  6. Select the name of the Council you have just created. This will allow the user to upload data files and to see their data results.
  7. You can create as many users for a particular Council as you like. Users can also have the rights to see the results for more than one Council; shift+click to multi-select Councils in the list.

Editing users

  1. Click on Admin | Users.
  2. Edit the name, email, password, or permissions.
  3. Save the user.

Editing a Council definition

  1. Click on Admin | All Councils to return to the homepage.
  2. Start to type the name of the Council in the textbox named 'Filter by Council name ...'
  3. When you see the name of the Council you are interested in, click on it.
  4. When you are viewing the Council, click on Admin | Edit Council
  5. Alter the values as you see fit: These values will take affect from the next Council data upload, manual or automatic.

Administrator and Normal Users

Administrator users can see data statistics for all Council areas. They can upload files in all Council areas. They can view land parcels across the State. They can add and remove and modify users and Councils.

Normal users have access to one or more Council areas. They can upload files in those areas, and can view land parcels in those Council areas only. They cannot make changes to other users or to Council configurations.

Manually Uploading Council Data Files

Prerequisite: LPI data for the new Council must be being provided in the daily EHC_LPMA_yyyymmdd.csv file.

A Council that has never had a file imported will look like this:

A Council can provide either:

  • A single file EHC_councilname_yyymmdd.csv that contains all data, or
  • Two files - EHC_councilname_LEP_yyyymmdd.csv with zoning data, and EHC_councilname_yyyymmdd.csv with all the other data.

If a Council is using the former, recommended, option:

  1. Click on Choose File
  2. When the file selector pops up, select the file you want to upload and click 'OK'.
  3. Click the large grey 'Upload data' button. A notice will appear that the import is being processed. Large Council areas may take quite some time to be processed the first time a file is uploaded. The user that uploaded the file will be emailed when the import is complete, with a summary of the results, or an email explaining why the file could not be processed.

For Councils that are using two files, there is one modification to the process: upload the LEP file before you upload the main data file.

Automatically Uploading Council Data Files

Once a Council can automatically produce a data file, it can be uploaded automatically using the LGA File Upload tool, which is separate to this project. For details on how to configure this, see the ePlanning team at the Dept of Planning & Infrastructure.

Understanding the results of a data file upload

There are three primary screens to understand:

  1. Data Summary
  2. Detailed Report
  3. Error Details

Tip: You will see reference to 'records' and to 'land parcels'. A record is a single line in the CSV file provided by a Council. A land parcel is a piece of land identified by a title reference (eg. 448//DP31875). In a perfect world, these would be the same, but they're not. Councils may, by design or by accident, have multiple records for a single land parcel. This is why you can see different numbers on pages that refer to records and pages that refer to land parcels.

Data Summary Screen

There are five main areas on this screen:

  1. Look up a land parcel by title reference or address
  2. A summary of records (not land parcels) currently stored for the Council.
  3. Detailed Reports for the last six imports. See the next section for more.
  4. The ability to import new data files, covered previously in this guide.
  5. Basic information on the last import - who, when, and how long.

Detailed Report Screen

This is the meat of the application, and vital for understanding the quality of a Council's data. These fields have the same meaning as the old manually-generated Data Quality Reports. Consult the user guide for those for detailed info.

The areas a Council staff member should be concentrating on are:

  • Reducing the number of 'Only In Council' records
  • Reducing the number of 'Only In LPI' records
  • Reducing the number of Invalid Land Parcels.

Note that this page primarily refers to land parcels, not to records. If there is a land parcel duplicated five times, then the data summary page will show 5 invalid records, whereas the Detailed Report page will show the number of land parcels, which is 1. The primary metric for Council data quality should be land parcels, not records.

Errors

This screen can be reached by clicking on the list of errors on either the data summary or detailed report pages. These are broken into categories:

  • Invalid Title Reference: Records that do not have an understandable title reference. This can be caused by a empty lot number (dp_lot_number), an empty plan label (dp_plan_number), or a plan label that does not being with one of DP or SP.
  • Duplicate DP: These are DP (deposited plan) records that have multiple records in the same Council file. This is generally an error.
  • Invalid Address: Records that are missing house numbers, street names, postcodes, etc.
  • Missing SI Zone: No information provided in the LEP_SI_Zone column, whether through the use of an LEP file or in the main data file.
  • Inconsistent Attributes: The Codes SEPP, as of May 2013, state that if any part of a strata plan is affected by a land-based exclusion, then all of it is. This section flags strata plans where lots within that plan have different zoning, or different land-based exclusions.

Searching for and viewing a land parcel

Administrators can search for land parcels across the State from the homepage. Searching from a Council's Data Summary page will restrict the search to that Council area.

Land parcels can be searched for by title reference or by address. To change the search from title reference to address, click the down-arrow in the 'title reference' button.

There is a known issue, where search by address is not as fast as it could be.

Clicking on a title reference will display the details of the land parcel. You can also see the common property information for a strata.
